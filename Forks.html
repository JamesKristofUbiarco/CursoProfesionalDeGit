    • Forks

    Cuando se quiere contribuir a un proyecto sin ser colaborador directo se hace un fork al proyecto original desde github, para que se cree una copia en nuestra cuenta de github y tengamos una copia actual del proyecto que queremos hacer cambios

    Para descargarlo en nuestro local se debe hacer el comando
    git clone <url>

    Todos los cambios que se hagan se harán en nuestro repositorio clonado en github, y si se quiere combinar con el del dueño se hace un pull request para que el lo acepte y se queden guardados finalmente los cambios

    Como nuestro repositorio local esta conectado a nuestro proyecto en github, este no se actualiza del original por lo que se debe crear un nuevo apuntador para poder descargar cambios del original, por lo que se crea un upstream hacia el repositorio original
    git remote add <upstream> <url>
    Y asi descargar siempre los cambios del original con git pull upstream master y una vez que se tengan los cambios actuales del original se actualiza hacia nuestro propio repositorio con git push origin master para que esté actualizado nuestro repositorio
